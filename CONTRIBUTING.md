We use a feature-branch workflow. Please do not commit directly to `master`.

* Create a branch for bug fixes: `git checkout -b fix/issue-description`

To keep our CHANGELOG clean, we follow [Conventional Commits](https://www.conventionalcommits.org/):
* `feat:` for new features.
* `fix:` for bug fixes.
* `docs:` for documentation changes.
* `style:` for formatting (no logic changes).
